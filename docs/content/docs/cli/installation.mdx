## Install via npm

```bash
# Global install (recommended)
npm install -g @supercheck/cli

# Or run without installing
npx @supercheck/cli health
```

<Callout type="info">
  Requires **Node.js 18** or later.
</Callout>

## Verify Installation

```bash
supercheck --version
supercheck health
```

## Docker

Run the CLI in a container using the official Node.js image:

```bash
docker run --rm \
  -e SUPERCHECK_TOKEN=$SUPERCHECK_TOKEN \
  node:20-slim sh -c "npm install -g @supercheck/cli && supercheck job trigger <job-id> --wait"
```

See [CI/CD Integration](./ci-cd) for Docker-based pipeline examples.

## Authenticate

The CLI requires a **CLI token** for API access. Create one in the dashboard:

1. Go to **Organization Admin → CLI Tokens**
2. Click **Create Token**
3. Copy the token (shown only once)

Then authenticate:

```bash
# Option 1: Environment variable (recommended for CI/CD)
export SUPERCHECK_TOKEN="sck_live_..."
supercheck login --token $SUPERCHECK_TOKEN

# Option 2: Direct login
supercheck login --token sck_live_a1b2c3d4...
```

## Self-Hosted Setup

Point the CLI at your self-hosted instance:

```bash
export SUPERCHECK_URL="https://supercheck.yourdomain.com"
export SUPERCHECK_TOKEN="sck_live_..."

supercheck login --token $SUPERCHECK_TOKEN --url $SUPERCHECK_URL
supercheck health
```

Standard Supercheck self-hosted deployments use Traefik + Let's Encrypt for automatic TLS — no certificate configuration required.

## Proxy Configuration

For corporate networks, the CLI respects standard proxy environment variables:

```bash
export HTTPS_PROXY="http://proxy.corp.com:8080"
export NO_PROXY="localhost,127.0.0.1,.internal.corp.com"
supercheck job trigger <id> --wait
```

## Initialize a Project

Scaffold a new `supercheck.config.ts` and `_supercheck_/` folder:

```bash
supercheck init
```

This creates:

```
my-project/
├── _supercheck_/
│   ├── playwright/   # Playwright test files
│   └── k6/           # k6 performance tests
└── supercheck.config.ts
```

The `init` command automatically:
- Installs `@playwright/test` and `@types/k6` as dev dependencies
- Downloads the Playwright **chromium** browser binary for local test execution
- Warns if **k6** is not installed (needed for performance tests)

## Test Dependencies

### Playwright (Browser Tests)

`supercheck init` handles Playwright setup automatically. If you need to install manually:

```bash
npm install --save-dev @playwright/test
npx playwright install chromium
```

### k6 (Performance Tests)

k6 is a standalone binary — install it via your system package manager:

```bash
# macOS
brew install k6

# Linux (snap)
sudo snap install k6

# Windows
choco install k6
# or
winget install k6
```

See [k6 installation docs](https://grafana.com/docs/k6/latest/set-up/install-k6/) for more options.

### Verify Dependencies

Run the doctor command to check that everything is set up:

```bash
supercheck doctor
```

## Uninstall

```bash
# Remove global install
npm uninstall -g @supercheck/cli

# Clear stored credentials
supercheck logout
```
