## Setup

1. Go to **Alerts → Add Provider**
2. Choose provider type
3. Enter credentials
4. Test the connection
5. Attach to monitors or jobs

## Threshold Logic

Prevent alert noise with consecutive failure thresholds.

### Example (Threshold = 3)

| Check | Status | Counter | Alert Sent |
|-------|--------|---------|------------|
| 1 | ❌ Fail | 1/3 | No |
| 2 | ❌ Fail | 2/3 | No |
| 3 | ❌ Fail | 3/3 | **⚠️ Yes** |
| 4 | ✅ Pass | Reset | **✅ Recovery** |

<Callout type="info">
Set threshold to 2-3 for most monitors to avoid false positives.
</Callout>

### Recovery Threshold

Configure how many consecutive passes before recovery alert:

| Setting | Description |
|---------|-------------|
| **Failure threshold** | Fails before alerting |
| **Recovery threshold** | Passes before recovery |

## Alert Types

| Type | When |
|------|------|
| **On Failure** | Monitor/test exceeds failure threshold |
| **On Recovery** | Status returns to normal |
| **SSL Expiry** | Certificate expires within N days |
| **Job Failure** | Scheduled job execution failed |
| **Job Success** | Scheduled job completed successfully |
| **Job Timeout** | Job exceeded execution time limit |
| **Scheduled** | Maintenance windows |

## Provider Configuration

### Email (SMTP)

```bash
SMTP_HOST=smtp.resend.com
SMTP_PORT=587
SMTP_USER=resend
SMTP_PASSWORD=your-key
SMTP_FROM_EMAIL=alerts@yourdomain.com
```

### Slack

1. Create [Incoming Webhook](https://api.slack.com/messaging/webhooks)
2. Copy webhook URL
3. Add in Supercheck

### Webhook

Send JSON payload to your endpoint:

```json
{
  "type": "failure",
  "monitor": {
    "id": "...",
    "name": "API Health",
    "url": "https://api.example.com/health"
  },
  "status": "down",
  "responseTime": 5000,
  "location": "us-east",
  "timestamp": "2024-01-15T10:30:00Z"
}
```

## Best Practices

- **Multiple channels** — Use Slack + Email for critical services
- **Escalation** — Start with Slack, email after 10 min
- **Grouping** — Avoid alert storms with thresholds
- **Testing** — Trigger test alerts before going live
