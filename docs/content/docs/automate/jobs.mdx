# Jobs

Group related tests and run them as a suite on a schedule or on-demand.

## What is a Job?

Jobs allow you to:
- Run multiple tests together
- Schedule recurring executions
- Create test suites
- Trigger from CI/CD

### Sequential Mode
Run tests one after another. Use when:
- Tests depend on each other
- Testing a workflow
- Resource constraints exist

### Parallel Mode
Run tests simultaneously. Use when:
- Tests are independent
- Faster execution needed
- Testing different features

## Creating a Job

```mermaid
sequenceDiagram
    participant U as User
    participant UI as Supercheck UI
    participant API as API Server
    participant DB as Database
    participant W as Worker

    U->>UI: Click Create Job
    UI->>U: Show Job Form
    U->>UI: Enter Job Details
    U->>UI: Select Tests
    U->>UI: Configure Schedule
    U->>API: POST /api/jobs
    API->>DB: Save Job Config
    DB-->>API: Job Created
    API-->>UI: Job ID
    UI-->>U: ✓ Job Created

    Note over W: When scheduled time arrives...
    W->>DB: Fetch Job Config
    DB-->>W: Job + Tests
    W->>W: Execute Tests
    W->>DB: Save Results
    W->>API: Send Alerts
    API-->>U: Notification
```

### Configuration Steps

<Steps>
  <Step>Navigate to **Jobs** in the sidebar</Step>
  <Step>Click **Create Job**</Step>

  <Step>Set up a schedule (optional)</Step>

```mermaid
    style A fill:#3b82f6,color:#fff
    style B fill:#8b5cf6,color:#fff
    style M fill:#f59e0b,color:#fff

```

Schedule jobs to run:
- **Hourly**: Every 1-24 hours
- **Daily**: At specific time
- **Weekly**: On specific days
- **Custom cron**: Advanced scheduling
- **On-demand**: Via API trigger
