# Quick Reference

Keep this page nearby for the actions you perform most often in Supercheck.

## Essential Shortcuts

| Action | How to do it | Where to find it |
| ------ | ------------ | ---------------- |
| **Run a test on demand** | Click the play icon next to any test | `Tests → All tests` |
| **Schedule a regression** | Open a job and set the cadence under **Schedule** | `Jobs → Job details` |
| **Create a monitor** | Use **Monitor from test** or **New monitor** | `Monitors` |
| **Share a report** | Copy the share link from the run drawer | Any run details view |
| **Mute alerts** | Toggle **Snooze** for the channel or monitor | Run details or monitor page |

<Callout type="tip">
  Hover over icons in the toolbar to reveal keyboard shortcuts for saving,
  running, and formatting tests.
</Callout>

## When Something Fails

<Steps>
  <Step>Open the run to review screenshots, console logs, and trace details.</Step>
  <Step>Check recent deploys or incident announcements for matching timestamps.</Step>
  <Step>Add a note with your findings so others see the same context.</Step>
  <Step>Re-run immediately or queue a follow-up job once the fix is ready.</Step>
</Steps>

## Helpful Views

- **Dashboard Overview**: Real-time pass rate, uptime, and alert volume
- **Run History**: Filter by tag, release, or owner to spot patterns
- **Variables & Secrets**: Rotate credentials without editing test code
- **Status Pages**: Share availability data with customers in seconds
