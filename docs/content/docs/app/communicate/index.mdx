## Smart Alert Thresholds

Avoid alert fatigue with consecutive failure thresholds. Only get notified when issues persist, not for transient blips.

| Check | Status | Failure Count | Alert? |
|-------|--------|---------------|--------|
| 1 | ❌ Fail | 1/3 | No |
| 2 | ❌ Fail | 2/3 | No |
| 3 | ❌ Fail | 3/3 | **Yes - Failure Alert** |
| 4 | ✅ Pass | Reset | **Yes - Recovery Alert** |

**Recommended settings:**
- **Failure threshold: 2-3** — Balances speed vs. false positives
- **Recovery threshold: 2** — Confirms service is truly stable

## Public Status Pages

Communicate service health to your users with branded status pages:

- **Component Management** — Organize services with individual status tracking and linked monitors
- **Incident Workflow** — Investigating → Identified → Monitoring → Resolved
- **Scheduled Maintenance** — Announce planned downtime with automatic status transitions
- **Subscriber Notifications** — Email, Slack, and webhook updates for incidents and status changes
- **Custom Branding** — Match your brand with logos, colors, and custom domains
- **90-Day Uptime History** — Visual uptime charts for transparency

## Use Cases

### For Internal Teams
- **On-call alerts** — Route critical failures to Slack and PagerDuty
- **Daily summaries** — Email digests of test results
- **CI/CD notifications** — Webhook integration with build pipelines

### For External Users
- **Service status** — Public page showing current system health
- **Incident updates** — Real-time communication during outages
- **Maintenance windows** — Advance notice of planned downtime
- **Subscriber notifications** — Automatic email updates for status changes
